Trial 2 of 12
🐦
Archen Says
"A Brachiosaurus this size needs to eat constantly — hundreds of kilograms of plants every single day. Your Brachiosaurus is getting hungry. It needs a hunger system before it can survive in this world."
🧑‍🏫 Parent Guide
▼
📦 What You're Building
A variable called Hunger that starts at 100 and slowly counts down. When it hits 0, it shows a warning message. Foundation for everything that follows.
⏱️ Time Estimate
30–45 minutes. Stopping after Step 4 is a valid win — the meter will exist and drain.
⚠️ What Might Be Hard
Variables are new. Key idea: a variable is just a number the game remembers. That's it.
🛠️ If Something Goes Wrong
Scratch has Ctrl+Z undo. Encourage Cole to notice what happened rather than immediately fixing it — that's real debugging.
🎮 Cole's Steps
1
Create the Hunger Variable
▼

This tells Scratch to start remembering a number called Hunger.

  1. Click your Brachiosaurus sprite in the sprite panel
  2. Click the orange Variables category
  3. Click "Make a Variable"
  4. Name it exactly: Hunger
  5. Make sure "For all sprites" is selected → click OK
💡 A Hunger counter will appear on the stage. That's the game remembering the number!
2
Set Hunger to 100 at the Start
▼
  1. Events (yellow) → drag out "when 🏁 clicked"
  2. Variables → drag "set Hunger to [ ]" under the flag
  3. Type 100 in the box
when 🏁 clicked set [Hunger] to (100)
3
Make Hunger Slowly Drain
▼
  1. Control → drag a "forever" loop under "set Hunger to 100"
  2. Inside the loop: Variables → "change Hunger by [ ]" → type -1
  3. Still inside: Control → "wait [ ] seconds" → type 1
when 🏁 clicked set [Hunger] to (100) forever change [Hunger] by (-1) wait (1) seconds end
▶️ Press the green flag — Hunger should count down from 100, one per second!
4
Stop It Going Below Zero
▼
  1. Inside the forever loop, add an "if/then" block (Control)
  2. Operators (green) → drag "[ ] < [ ]" into the hexagon slot
  3. Left box: drag the Hunger variable oval · Right box: type 1
  4. Inside the if/then: Variables → "set Hunger to 0"
when 🏁 clicked set [Hunger] to (100) forever change [Hunger] by (-1) wait (1) seconds if (Hunger) < (1) then set [Hunger] to (0) end end
5
Add a Warning When Hunger Gets Low
▼
  1. Below "set Hunger to 0" (still inside the if/then), add Looks → "say [ ] for [ ] seconds"
  2. Type: "🦕 Hungry! Find food!" · Seconds: 2
▶️ Temporarily set starting Hunger to 5 to test it faster — then put it back to 100!
"The hunger system is live. The Brachiosaurus feels time passing now. This is what makes a survival game real — the world doesn't wait." — Archen
🏅
Trial 2 Complete!
The Hunger system is alive. On to Trial 3!

Trial 3 of 12
🐦
Archen Says
"You've given the Brachiosaurus a hunger — now it needs something to actually EAT. In the wild, Brachiosaurus spent most of their day munching on treetop leaves. Let's put some food in this world and teach your dino to find it!"
🧑‍🏫 Parent Guide
▼
📦 What You're Building
A food sprite that gets eaten when the Brachiosaurus touches it, restores Hunger, and reappears somewhere new. This is the core gameplay loop.
🆕 New Concepts

⚠️ What Might Be Hard

🚀 If Cole Finishes Early
Ask: "What if the dino is already full at 100 and eats food? Should it go above 100?" — natural bridge to Trial 4!
🎮 Cole's Steps
1
Make a Food Sprite
▼
  1. Bottom-right of Scratch → click Add Sprite (cat face button)
  2. Choose from the library or click the paintbrush to draw your own leaf/fern
  3. Name the sprite "Food"
🎨 A simple green blob works great — Brachiosaurus wasn't picky!
2
Write the Setup Script
▼

Click your Food sprite so you're editing its scripts.

when 🏁 clicked go to x: (0) y: (0) show
3
Write the Eating Script
▼

Still on the Food sprite — add a second script:

when 🏁 clicked forever if <touching [Brachiosaurus]?> then change [Hunger] by (20) go to (random position) end end
💡 Every frame, Food checks: "Is the dino touching me?" If yes — add 20 Hunger and teleport somewhere new, like leaves growing on a new tree!
4
Test It!
▼
  1. Press the green flag 🏁
  2. Move your Brachiosaurus into the Food
  3. Hunger should go up by 20 · Food should jump to a new spot
🔧 Hunger not going up? Check that Hunger was made "For all sprites" in Variables.
🔧 Food not moving? Make sure go to random position is inside the if block.
5
Optional Polish — Make It Feel Real ⭐
▼

After change [Hunger] by (20), replace go to random position with:

hide wait (0.3) seconds show go to (random position)
✨ The leaf vanishes then reappears — feels much more satisfying to eat!
"YES! Now THAT'S a Brachiosaurus who knows how to eat! Your dino has a hunger AND a way to fix it. That's a real game loop." — Archen

Trial 4 of 12
🐦
Archen Says
"Right now, if your dino eats too much, Hunger just keeps climbing forever. And when it hits zero... nothing really happens. Let's fix both of those. A real survival game needs real limits — and real consequences."
🧑‍🏫 Parent Guide
▼
📦 What You're Building
Two things: (1) a cap so Hunger can't go above 100, and (2) a visible danger state — the screen flashes or Archen yells — when Hunger hits zero. This makes the stakes feel real.
⏱️ Time Estimate
20–30 minutes. Short, satisfying win.
⚠️ What Might Be Hard
The Hunger cap goes on the Food sprite script (where Hunger gets added), not on the Brachiosaurus. Cole may need prompting to find the right script.
🎮 Cole's Steps
1
Cap Hunger at 100
▼

Click your Food sprite. Find the eating script (the forever loop). After change [Hunger] by (20), add:

if (Hunger) > (100) then set [Hunger] to (100) end
💡 Now even if the dino eats when almost full, Hunger stops at exactly 100.
2
Add a Danger State at Zero
▼

Click your Brachiosaurus sprite. Find the forever loop from Trial 2. Inside the if Hunger < 1 block, add below "set Hunger to 0":

set [color v] effect to (50) wait (0.2) seconds set [color v] effect to (0) say [🚨 STARVING! Find food NOW!] for (2) seconds
✨ The dino flashes red and Archen panics. Now zero feels dangerous!
💡 The "color effect" blocks are under Looks. The number 50 gives a red-ish tint — Cole can experiment with different numbers!
3
Test Both Changes
▼
  1. Set starting Hunger to 5 temporarily — watch the danger flash happen
  2. Set it to 95 and eat food — check Hunger stops at 100, not 115
  3. Set Hunger back to 100 when done
"Now the dino knows what danger feels like. The game has tension. That's the secret ingredient." — Archen

🎮 Cole's Steps
1
Clear Old Movement (If Any)
▼

Click your Brachiosaurus sprite. If you already have arrow key movement blocks, right-click them and choose Delete. We're replacing them with something better.

💡 If you don't have any movement yet — perfect, skip straight to Step 2!
2
Build the Movement Script
▼

On your Brachiosaurus sprite, build this script. Use Sensing for the key blocks, Motion for movement, and Control for the if blocks:

when 🏁 clicked forever if <key [right arrow] pressed?> then change x by (5) point in direction (90) end if <key [left arrow] pressed?> then change x by (-5) point in direction (-90) end if <key [up arrow] pressed?> then change y by (5) point in direction (0) end if <key [down arrow] pressed?> then change y by (-5) point in direction (180) end end
🎮 Cole's choice: The number 5 is the speed. Want a faster dino? Try 8 or 10. Want slow and majestic? Try 3. It's your game!
3
Stop the Dino Flipping Upside Down
▼

When the dino moves left, it might flip upside down. Fix it:

  1. Click the little ℹ️ info button on the Brachiosaurus sprite (or look above the stage)
  2. Find the rotation style buttons — three small icons
  3. Click the middle one (left-right only)
✨ Now the dino flips left/right but never goes upside down!
"Look at it go! Smooth, powerful, responsive. That's a dinosaur worth controlling." — Archen

🧑‍🏫 Parent Guide

⚠️ What Might Be Hard
The speed variable approach requires updating the movement script from Trial 5. Walk Cole through finding the right script carefully.
🎮 Cole's Steps
1
Make a Speed Variable
▼
  1. Variables → "Make a Variable"
  2. Name it: Speed
  3. "For all sprites" → OK

Now add this to the beginning of your Brachiosaurus's hunger script (right after when 🏁 clicked):

set [Speed] to (5)
💡 This sets the normal walking speed to 5. We'll change this number when the dino enters water!
2
Update Movement to Use Speed Variable
▼

Go to your movement script from Trial 5. Replace the number 5 in all four directions with the Speed variable oval:

if <key [right arrow] pressed?> then change x by (Speed) point in direction (90) end if <key [left arrow] pressed?> then change x by ((Speed) * (-1)) point in direction (-90) end if <key [up arrow] pressed?> then change y by (Speed) point in direction (0) end if <key [down arrow] pressed?> then change y by ((Speed) * (-1)) point in direction (180) end
💡 For left and down, we need Speed * -1 to go the negative direction. Find the multiply block in Operators!
3
Add the Wading Effect to the Water Script
▼

Click your Water sprite. Update its script to handle entering AND leaving water:

when 🏁 clicked forever if <touching [Brachiosaurus]?> then set [Speed] to (2) set [color v] effect to (30) of [Brachiosaurus v] change [Thirst] by (15) if (Thirst) > (100) then set [Thirst] to (100) end else set [Speed] to (5) set [color v] effect to (0) of [Brachiosaurus v] end end
🌊 In water: speed drops to 2 (slow wading) and the dino turns blue. Out of water: speed returns to 5 and colour resets. The else block handles the "not in water" state!
💡 To set an effect on a different sprite, use Looks → "set [effect] effect to [ ] of [sprite]" — look for it carefully in the Looks section!

Coming Next — Trial 11
Game Over screen! When Hunger or Thirst hits zero, everything stops. Real stakes, real ending.
Trial 11 of 12
🐦
Archen Says
"Right now, when Hunger or Thirst hits zero... the game just keeps going. That's not a survival game — that's a screensaver. We need a proper Game Over. When the dino can't survive anymore, the game knows it, stops everything, and tells the player. This is the most important trial."
🧑‍🏫 Parent Guide
▼
📦 What You're Building
When Hunger OR Thirst hits zero, a "Game Over" broadcast stops all scripts and shows a final message with the score. This uses Scratch's broadcast system for the first time.
🆕 New Concepts
Broadcasts — Scratch's way of sending a signal to all sprites at once. broadcast [Game Over] and when I receive [Game Over]. This is the most powerful communication tool in Scratch.
⏱️ Time Estimate
25–35 minutes. This trial has the most emotional payoff — the first time the game truly ends.
⚠️ What Might Be Hard
The stop [other scripts in sprite] block is easy to confuse with stop [all]. Using stop [all] is actually fine here and simpler — recommend it.
🎮 Cole's Steps
1
Create a Game Over Broadcast
▼

We need a way to tell ALL sprites "the game is over." In Scratch, we do this with a broadcast.

  1. Go to Events (yellow blocks)
  2. Find the "broadcast [ ]" block
  3. Click the dropdown → "New message"
  4. Name it: Game Over
💡 A broadcast is like an announcement over a loudspeaker — every sprite in the game hears it at the same time!
2
Trigger Game Over When Stats Hit Zero
▼

Click your Brachiosaurus sprite. Find the forever loop with the Hunger/Thirst checks. Update both if blocks to broadcast Game Over:

if (Hunger) < (1) then set [Hunger] to (0) broadcast [Game Over v] end if (Thirst) < (1) then set [Thirst] to (0) broadcast [Game Over v] end
💡 Remove the old warning messages from these blocks — Game Over replaces them!
3
Make the Brachiosaurus React
▼

Still on the Brachiosaurus sprite, add a brand new script:

when I receive [Game Over v] say (join [Game Over! Score: ] (Score)) for (999) seconds stop [all v]
💡 join is in Operators — it glues two pieces of text together. Drag the Score variable oval into the second slot!
💡 stop [all] is in Control — it freezes every script in the whole game instantly.
4
Test the Game Over Screen
▼
  1. Set Hunger to 3 temporarily and press green flag 🏁
  2. Wait — the dino should call Game Over, show your score, and freeze
  3. Everything stops: movement, hazard, score climbing — all of it
  4. Put Hunger back to 100
"Your game has an ending. A real ending with your score shown. That's not a toy anymore — that's a complete game. I'm genuinely impressed." — Archen
